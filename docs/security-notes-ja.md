# セキュリティに関する注意事項

## 制限付きビューアはDRMではありません

MD//WORKS 制限付きビューアフォーマット v1 は、一般的なコピー、選択、ドラッグ、コンテキストメニュー、印刷を抑制することはできますが、スクリーンショット、カメラ撮影、ブラウザの開発者ツール、メモリ検査、またはHTML/ランタイムの改ざんを防ぐことはできません。

## 表示データは機密ではありません

表示ボックスのAESキーは、生成されたHTML内に保存されます。表示の暗号化は、ソース内に表示用HTMLを明白な平文として残すことを避けるためのものであり、意図的にコンテンツを抽出しようとする閲覧者から隠すものではありません。

## 復元データはパスワードの強度に依存します

復元用ペイロードは、PBKDF2-HMAC-SHA256を介して復元パスワードから導出されたキーを使用し、AES-256-GCMで暗号化されます。弱いパスワードは、オフラインでの推測攻撃に対して脆弱なままです。ユーザーは、長く、使い回しのないパスフレーズを選択し、安全に保管する必要があります。

## パスワードの取り扱い要件

ジェネレーターは、正確なパスワード文字列をそのままUTF-8エンコーディングとPBKDF2に渡さなければなりません。以下を行ってはなりません：

* 空白のトリミング
* 改行の追加や削除
* 大文字・小文字の変換
* Unicode正規化の適用
* 類似文字の置換
* 全角または半角文字の変換

## Markdownの保持

ジェネレーターは、復元用ペイロードに保存する前に、Markdownを自動的にUnicode正規化すべきではありません。互換性テストでは、バイト単位の完全一致とは別に、NFC正規化されたテキストを比較する場合があります。

## 暗号化の実装

AES、GCM、PBKDF2、または乱数生成を独自に手動で実装しないでください。必要に応じて `cryptography` のレビュー済みライブラリ機能や、Pythonの標準の安全な乱数APIを使用してください。

## リポジトリの衛生管理

実際のパスワード、シークレット、プライベートな文書、または機密性のあるソース画像をコミットしないでください。サンプルには無害なダミーデータのみを使用する必要があります。
